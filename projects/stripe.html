My Role

As a member of the Agent Platform team, I worked on Stripe's ticket resolution platform, Case Studio. This platform is the agent-facing side of the helpdesk, where agents are able to interact with customers to assist with questions and resolve help tickets.

During my internship, I had the oppportunity to work on two different projects: Case Events and Agent Satisfaction. I also gained experience in code reviews, project scoping, and cross-team collaboration.

Project 1: Case Events

Agents often pass tickets off between each other, especially in ticket escalation. They should be able to easily see what decisions have been already made on the ticket.

Summary

  • Added a data collection and display point into Case Studio which allows agents to see a log of what changes have been made to the case by themselves or by other agents
  • Collected data on ~10k daily case changes and stored it in a MongoDB model
  • Integrated data with existing case messages to create a visual timeline of a case's history

Project 2: Agent Satisfaction (ASAT)

As developers of Case Studio, my team had no concrete and regular way to receive unbiased feedback from agents about how well Case Studio is working for them.

Summary

  • Scoped out new project including timeline and Jira tasks
  • Worked with adjacent teams and UI designer to determine the best questions to ask agents
  • Created a 5-question survey displayed at pseudo-random intervals, once every calendar month
